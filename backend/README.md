# Deep Researcher Directory Structure

The directory structure of this project is organized as follows:

```
backend/
│   .gitignore
│   .python-version
│   main.py {server entry point, run as `uv run ./main.py}
│   pyproject.toml
│   README.md
│   uv.lock
│
├───.venv/ {virtual environment files}
│
├───idea/ {If any quick ideas or notes, I can put them here.}
│       Ideas.md
│
└───main {main internal source code for the project}
    │   __init__.py
    │
    └───src {The core and logic of the application goes here in an organised manner.}
        │   apis.py
        │   README.md
        │   __init__.py
        │
        ├───llms [All the LLM related code and logic goes here, including the API calls, response handling, etc.]
        │   │   README.md
        │   │   __init__.py
        │   │
        │   ├───gemini [All the code related to Gemini LLM goes here, including API calls, response handling, etc.]
        │   │       README.md
        │   │       __init__.py
        │   │
        │   └───ollama [All the code related to Ollama LLM goes here, including API calls, response handling, etc.]
        │           README.md
        │           __init__.py
        │
        ├───research [All the code related to research and data gathering goes here, including web scraping, data processing, etc.]
        │       README.md
        │       __init__.py
        │
        ├───store [All the code related to data storage and management goes here, including database interactions, file storage, etc.]
        │   │   DBManager.py
        │   │   README.md
        │   │   __init__.py
        │   │
        │   ├───bucket [All file storage management]
        │   │       README.md
        │   │
        │   └───database [All the databases will be autogenerated inside this repo.]
        │           basicUtils.db.sqlite3
        │           history.db.sqlite3
        │           README.md
        │           scrapes.db.sqlite3
        │
        ├───tools [All code related to create the MCP client goes here, including the client logic, API calls, response handling, etc.]
        │       README.md
        │       __init__.py
        │
        └───utils [All the utility functions and helper code goes here, including common functions, error handling, etc.]
                README.md
                __init__.py
```



> There are README.md files in each subdirectory to provide detailed documentation and explanations for the code and logic contained within those directories. Please refer to those README.md files for more information on the specific code and logic implemented in each directory.
